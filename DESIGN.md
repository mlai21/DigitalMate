# DESIGN.md — DigitalMate 设计系统

面向设计/编码 agent 的设计规范。构建任何 UI（聊天界面、管理后台）前先读本文件，保持视觉一致。

产品背景见 [docs/prd.md](docs/prd.md)：DigitalMate 是一个有人设、有记忆的私人数字伙伴。UI 的使命是让用户感觉在和一个「人」聊天，而不是在操作一个工具。

---

## 1. 视觉主题与氛围（Visual Theme & Atmosphere)

**关键词：温暖陪伴、安静克制、像一间有阳光的客厅。**

- 整体气质是「温暖极简」：暖白画布、柔和圆角、低对比的暖灰文字，没有冷冰冰的科技蓝和玻璃拟态。
- 界面密度低，留白慷慨。聊天界面是绝对主角，其余一切（导航、设置）都退后。
- 动效轻柔缓慢（200–300ms ease-out），像人的呼吸节奏，拒绝弹跳和炫技动画。
- 管理后台可以信息密度稍高，但延续同一套暖色语言，不切换成冷色仪表盘风格。
- 两套主题：浅色为默认（暖白），深色为夜间（暖炭），色温都偏暖。

## 2. 色彩与角色（Color Palette & Roles）

### 浅色主题（默认）

| 语义名 | Hex | 角色 |
|---|---|---|
| `canvas` | `#FAF7F2` | 页面背景，暖白纸感 |
| `surface` | `#FFFFFF` | 卡片、对方气泡、输入框背景 |
| `surface-sunken` | `#F3EEE6` | 侧栏、分区背景、代码块底 |
| `border` | `#E9E2D6` | 分割线、卡片描边（1px） |
| `ink` | `#2D2A26` | 主文字，暖近黑（禁用纯黑） |
| `ink-secondary` | `#6E675F` | 次要文字、时间戳、说明 |
| `ink-faint` | `#A39B90` | 占位符、禁用态文字 |
| `accent` | `#E8684A` | 主 CTA、发送按钮、链接、选中态（珊瑚橙） |
| `accent-hover` | `#D6573A` | accent 的 hover/active |
| `accent-soft` | `#FBEAE4` | 用户气泡背景、accent 的浅色衬底 |
| `success` | `#4E9B6F` | 成功状态、在线指示 |
| `warning` | `#DB9B2D` | 警告、待确认（如 Skill 待审批） |
| `danger` | `#CF4B3E` | 错误、删除操作 |

### 深色主题

| 语义名 | Hex | 角色 |
|---|---|---|
| `canvas` | `#201D19` | 页面背景，暖炭色 |
| `surface` | `#2B2723` | 卡片、对方气泡 |
| `surface-sunken` | `#171512` | 侧栏、分区背景 |
| `border` | `#3B362F` | 分割线、描边 |
| `ink` | `#F0EAE1` | 主文字，暖米白（禁用纯白） |
| `ink-secondary` | `#A79E93` | 次要文字 |
| `accent` | `#F07B5C` | 主 CTA（比浅色主题略提亮） |
| `accent-soft` | `#453029` | 用户气泡背景 |

**规则**：accent 是唯一的品牌色，一屏内只用于一个主操作；大面积色块一律用中性暖色，彩色只做点缀。状态色（success/warning/danger）只用于状态反馈，不做装饰。

## 3. 字体规则（Typography Rules）

**字体栈**（不引入网络字体，用系统栈保证中文渲染与加载速度）：

```css
--font-sans: -apple-system, BlinkMacSystemFont, "PingFang SC",
             "HarmonyOS Sans SC", "Noto Sans SC", "Segoe UI", sans-serif;
--font-mono: ui-monospace, "SF Mono", "JetBrains Mono", Menlo, monospace;
```

| 层级 | 字号/行高 | 字重 | 用途 |
|---|---|---|---|
| Display | 28px / 1.3 | 600 | 欢迎页标题、空状态大字 |
| Title | 20px / 1.4 | 600 | 页面标题、对话名称 |
| Section | 16px / 1.5 | 600 | 卡片标题、设置分组 |
| Body（聊天正文） | 16px / 1.75 | 400 | 聊天气泡文字（中文行高必须 ≥1.7） |
| Body-sm | 14px / 1.6 | 400 | 后台表格、次要正文 |
| Caption | 12px / 1.5 | 400 | 时间戳、字数统计、辅助说明 |
| Mono | 13px / 1.6 | 400 | 代码块、工具调用日志（仅后台） |

**规则**：正文字重只用 400/600 两档；中文场景不用 letter-spacing 负值；聊天气泡内文字永远 16px，不因屏幕大小缩小。

## 4. 组件样式（Component Stylings）

### 聊天气泡（核心组件）

- **对方（DigitalMate）气泡**：`surface` 底 + 1px `border` 描边，圆角 `18px`，靠左；头像（圆形 36px）常驻在侧。
- **用户气泡**：`accent-soft` 底、无描边，圆角 `18px`，右下角收小为 `6px`（指向发送者），靠右，无头像。
- 气泡内边距 `12px 16px`；连续多条消息间距 `6px`，不同发言人之间 `20px`。
- **输入中指示**：三个 `ink-faint` 圆点柔和起伏（1.4s 循环）——这是「对方在打字」的真人感信号，**永远不用 spinner**。
- 分段发送的多条消息逐条淡入上移（fade + 8px translateY，250ms）。

### 输入栏

- 底部悬浮，`surface` 底、圆角 `24px`、1px `border`，聚焦时描边变 `accent`（无 glow）。
- 发送按钮：36px 圆形 `accent` 底白色图标；无内容时降为 `ink-faint` 禁用态。
- 多行自动增高，最多 6 行后内部滚动。
- 输入栏增高时，消息列表必须按输入栏实时高度预留底部空间，禁止覆盖最后一条消息。

### 按钮

| 类型 | 样式 | 用途 |
|---|---|---|
| Primary | `accent` 底、白字、圆角 `12px`、高 40px | 一屏一个主操作 |
| Secondary | `surface` 底、1px `border`、`ink` 字 | 次要操作 |
| Ghost | 无底无边、`ink-secondary` 字，hover 变 `surface-sunken` 底 | 工具栏、行内操作 |
| Danger | 默认 Ghost 形态 `danger` 字色，确认弹层内才用 `danger` 实底 | 删除记忆等 |

状态：hover 加深底色约 8%；active 缩放 `scale(0.98)`；禁用 40% 不透明度。不用渐变底、不用阴影按钮。

### 卡片（后台：记忆条目、Skill、反思记录）

- `surface` 底、1px `border`、圆角 `16px`、内边距 `20px`；hover 时描边变深一档，**不加阴影不上浮**。
- 待确认状态（Skill 草稿、行为修正建议）：左侧 3px `warning` 竖条 + 角标。

### 导航

- Web 端：左侧栏 `surface-sunken` 底，宽 260px；条目圆角 `10px`，选中态 `accent-soft` 底 + `accent` 字。
- 移动端：聊天页全屏，后台入口收进左上角抽屉。

### 表单与反馈

- 输入框/选择器：与输入栏同语言（圆角 12px、聚焦 accent 描边）。
- 开关（主动性档位等）：选中 `accent` 底；带文字标签，不裸放。
- Toast：顶部居中滑入，`ink` 深底白字（浅色主题下反色），圆角 `12px`，3s 自动消失。

## 5. 布局原则（Layout Principles）

- **间距标尺**：4px 基准 —— `4 / 8 / 12 / 16 / 20 / 24 / 32 / 48 / 64`。
- **聊天列**：内容最大宽度 `720px` 居中；气泡最大宽度为列宽的 78%。
- **后台**：内容最大宽度 `1080px`；卡片网格间距 `16px`。
- 页面级留白：移动端左右 `16px`，桌面端 `32px`。
- 圆角体系：小件 `10–12px`，卡片/气泡 `16–18px`，输入栏/弹层 `20–24px`——整体偏圆，传达柔和。
- 一屏只有一个视觉焦点；聊天页的焦点永远是最新消息与输入栏。

## 6. 深度与层级（Depth & Elevation）

阴影极度克制，暖色调、低透明度，**默认平面，仅浮层有影**：

| 层级 | 样式 | 用途 |
|---|---|---|
| Level 0 | 无阴影，靠 `border` 和底色区分 | 卡片、气泡、侧栏 |
| Level 1 | `0 2px 8px rgba(60, 45, 30, 0.06)` | 悬浮输入栏、下拉菜单 |
| Level 2 | `0 8px 32px rgba(60, 45, 30, 0.12)` | 弹窗、抽屉 |

弹窗遮罩：`rgba(32, 29, 25, 0.4)` + 轻微 backdrop-blur（4px）。深色主题下阴影同层级但透明度加倍。

## 7. 应该与不应该（Do's and Don'ts）

**Do：**

- 聊天等待时用「打字中」三点指示，让等待像真人在输入。
- 空状态用 DigitalMate 的口吻写文案（如「今天想聊点什么？」），保持人设语气。
- 时间戳、系统提示等元信息用 `caption` 级灰字，弱化存在感。
- 深浅两套主题都保持暖色温。
- 破坏性操作（删除记忆、清空数据）必须二次确认，确认按钮才是 `danger` 实底。

**Don't：**

- 不用纯黑 `#000` / 纯白 `#FFF` 做大面积文字或背景（暖色近似值替代；`surface` 的白仅用于小面积卡片）。
- 不用科技蓝、紫色渐变、玻璃拟态、霓虹光效——这是「伙伴」不是「赛博工具」。
- 不在聊天界面展示任何推理过程、工具调用、token 数（PRD 红线：不暴露思考过程；这些只进后台日志页）。
- 不用 spinner、进度条、骨架屏出现在对话流里。
- 不用弹跳（bounce/spring 过冲）动画；不给卡片 hover 上浮加影。
- emoji 由人设按习惯在文字里使用，UI 装饰本身不堆砌 emoji。

## 8. 响应式行为（Responsive Behavior）

| 断点 | 范围 | 布局 |
|---|---|---|
| Mobile | < 768px | 单列全屏聊天；侧栏收为抽屉；输入栏贴底 |
| Tablet | 768–1079px | 侧栏可折叠；聊天列居中 |
| Desktop | ≥ 1080px | 侧栏常驻 + 聊天列 720px 居中 |

- 移动优先开发；触控目标最小 `44×44px`。
- 移动端适配安全区（`env(safe-area-inset-bottom)`，输入栏尤其注意）。
- 字号不随断点缩放，聊天正文恒定 16px（同时避免 iOS 聚焦自动放大）。
- 键盘弹起时输入栏跟随上移；仅当用户原本位于会话底部附近时继续跟随最新消息。用户正在阅读历史内容时不得强制滚动，应显示「新消息」按钮供用户主动跳转。

## 9. Agent 提示指南（Agent Prompt Guide）

**快速色值参考（浅色主题）**：背景 `#FAF7F2`、卡片 `#FFFFFF`、主文字 `#2D2A26`、次文字 `#6E675F`、描边 `#E9E2D6`、品牌色 `#E8684A`、用户气泡 `#FBEAE4`。

**生成 UI 时的基准提示词**：

> 使用 DigitalMate 设计系统：暖白背景（#FAF7F2）、暖近黑文字（#2D2A26）、唯一品牌色珊瑚橙（#E8684A）只用于主操作；系统中文字体栈，聊天正文 16px / 行高 1.75；大圆角（气泡 18px、卡片 16px）、1px 暖灰描边、默认无阴影；间距用 4px 标尺；动效 200–300ms ease-out，克制柔和。整体氛围是温暖的私人伙伴，不是冷色调科技产品。

**典型任务示例**：

- 「按 DESIGN.md 构建聊天主界面：左侧可折叠会话列表，中间 720px 聊天列，底部悬浮圆角输入栏，对方消息带打字中三点指示。」
- 「按 DESIGN.md 构建后台记忆管理页：卡片网格展示记忆条目，待确认条目左侧 warning 竖条，删除需二次确认。」

**冲突处理**：本文件与通用审美习惯冲突时以本文件为准；本文件与 [docs/prd.md](docs/prd.md) 的产品红线冲突时以 PRD 为准。
