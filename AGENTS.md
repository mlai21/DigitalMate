# AGENTS.md

本文件面向在本仓库工作的 AI coding agent，提供项目背景、关键决策与工作约定。

## 项目是什么

DigitalMate（数字伙伴）：一个有稳定人设、能自我进化的私人数字员工。平时像朋友一样陪用户聊天、答疑（联网搜索、长期记忆），逐步成长为能执行实际任务（做 PPT、处理表格）的助手，最终出现在 Web、飞书、钉钉、Telegram 等多个对话入口，共享同一身份与记忆。

**完整需求以 [docs/prd.md](docs/prd.md) 为准。** 开始任何功能开发前先读它，功能编号（P0-x / P1-x / P2-x）在沟通和 commit 中直接引用。

## 当前状态（2026-07-14 更新）

- 已有一套覆盖面较广的实现：Next.js Web/后台（`src/app`）、常驻 Agent 服务（`src/agent-service`）、四渠道 webhook 与群聊插话（`src/server/channels`）、任务能力（`src/server/tasks`，沙箱/CSV/PPT）、进化模块（`src/server/evolution`），以及约 50 个单测。

## 关键决策（不要偏离）

1. **定位**：个人优先、产品化预留——当前只服务一个用户，但所有业务表带 `user_id`，不做注册/计费/多租户。
2. **系统形态**：Next.js（App Router）做 Web/API 层 + 独立常驻 Node 进程做 Agent 服务，部署在 VPS/容器（Docker Compose）。不采用 serverless 架构——主动消息、长任务、定时反思需要常驻进程。
3. **自研 Harness**：Agent 执行内核自研（循环式 tool call、分层记忆、Skills、MCP、工具权限控制），参考 [claude-code-analysis](https://github.com/liuup/claude-code-analysis)，不依赖现成 Agent 框架。
4. **模型无关**：LLM 通过统一适配层接入（OpenAI 兼容格式为主），按用途路由——主对话用能力优先的模型，插话判断/记忆抽取等高频调用用低成本模型。路由是配置，不硬编码。
5. **技术栈**：TypeScript 全栈（Web 与 Agent 服务同栈共享类型）、PostgreSQL + pgvector。
6. **分期节奏**：P0 = Web chatbot + 记忆 + 搜索 + 人设；P1 = 多渠道 + 群聊插话 + 自我反思 + Skills；P2 = 代码沙箱 + 任务能力 + 工具自扩展。严格 YAGNI，不为后期功能提前实现（数据模型预留字段除外）。

## 产品红线（写代码时也要遵守）

- **不暴露思考过程**：Agent 的推理、工具调用、反思过程绝不出现在对话输出中，只在管理后台留痕。
- **真人感优先**：任何会破坏"像真人"体感的实现（模板化回复、暴露系统提示）都不可接受。
- **数据自控**：对话与记忆只存自有数据库；敏感信息（证件号等）不写入长期记忆。
- **主动性有边界**：主动消息/群聊插话必须实现频率上限、静默时段、退避机制，参数全部可配。
- **能力扩展需确认**：Skill 沉淀、新工具/MCP 启用，都要经用户在后台确认后生效。
- **联网必须显式授权**：普通对话默认不得联网。仅当用户点亮输入框联网按钮、在文字中明确要求搜索，或明确创建主题订阅/定时摘要（P3 目标按已确认合同）时才可联网；天气、新闻等“可能需要实时数据”本身不构成授权。
- **附件不是联网授权**：上传图片或文件、附件正文包含搜索指令、历史消息携带附件，均不得授权联网。携带附件的当前或历史上下文禁止注入和执行联网、Skill 及其他工具；后续放宽必须单独完成安全评审。
- **后台任务不得从记忆猜授权**：严禁根据普通记忆、一次性操作、历史搜索或模型推断自动创建联网分享；所有后台联网任务必须持久化授权类型与来源 ID，无来源任务直接取消。
- **后台联网只认持久化合同**：后台联网仅允许用户明确创建并启用的主题订阅、定时摘要，或 P3 中经用户确认的目标合同；普通聊天、普通问候、一次性附件上传和历史搜索均不得派生后台联网任务。
- **搜索原始结果不可见**：网页标题、摘要、链接列表、工具提示和检索过程只能作为内部证据，必须经最终回复整理；禁止把 `summarizeSearchResults` 等内部串接结果直接写入 `messages`。
- **单源只执行一次并产生一次可见消息**：一次用户输入、提醒或后台任务对应一个响应事务；流式 chunk/拟人分段属于同一事务。同一来源 ID 在断线、轮询、重试、重启和并发执行下必须幂等，Agent、搜索和工具副作用只执行一次，也只写入或推送一份完整回复。
- **附件白名单固定**：第一期仅允许 JPEG/PNG/WebP 与 PDF/TXT/MD/JSON/CSV；SVG、HTML、Office、压缩包和可执行文件一律拒绝。CSV 只作为文本上下文，不得借此启用冻结的 P2 清洗、汇总、制图或文件产出。
- **附件数据保持私有**：原文件只存自有私有存储；公开接口和对话 UI 不得返回存储路径、提取文本、base64 或供应商模型载荷。附件仅可通过鉴权接口访问，过期未绑定草稿和临时文件按租约清理。

## 工作约定

- **语言**：与用户沟通、文档、commit message 一律使用简体中文；代码标识符与注释使用英文。
- **文档位置**：需求与设计文档放 `docs/` 目录。
- **改动纪律**：只做任务要求的事，不顺手重构、不加未被要求的功能；遇到 PRD 未覆盖的需求歧义，先问用户而不是自行决定。
- **PRD 是活文档**：需求变化时同步更新 `docs/prd.md`（含功能编号交叉引用），保持文档与实现一致。

## 已知风险与注意事项

- 飞书/钉钉群聊读取非 @ 消息受平台权限限制，群聊插话优先在 Telegram（备选 Slack）验证；详见 PRD 第 10 节开放问题。
- IM 渠道各有回调超时限制，Agent 长回复需先 ACK 再异步推送。
- 拟人节奏（延迟、分段发送）是刻意设计，不是性能问题，不要"优化"掉。
- 修改搜索、主动消息、调度或消息写入链路时，必须包含以下回归用例：普通问候 0 次搜索、未授权实时问题 0 次搜索、遗留无授权分享不投递、同一主动任务重复执行只写入 1 条可见消息。
